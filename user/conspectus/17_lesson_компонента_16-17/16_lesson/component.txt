- Компонента - это функция, которая возвращает разметку JSX. Это кусок какого-то html.
Компонента - это тег. 
- Каждая компонента вызывается всегда с параметрами, и самый главный параметр это props.
Реакт при вызове функции всегда засовывает в передаваемый параметр объект, который попадает внутрь параметра
props. То есть находясь внутри функции мы можем этим пропсом манипулировать, считывать его значение, смотреть
что там в props к нам пришло извне.
Пример функции:
const Header = (props) => {
    return (div 
    <a>it-incubator.by</a>
    <a>repetitora.net</a>
    <a>it-kamasutra.net</a>
    div);
}

Вот эта компонента Header всегда возвращает эту разметку. А тут херяк меня вызвали и дали еще какой-то объект 
props, которым я могу играться. Под который я могу подстроиться и разметку немножко отобразить по-разному.

 Например:
 let obj = {
    name: 'Dima'
 }

 Header(obj);

 - Компонента Header вызывается, и вот этот объект obj , у которого есть ключ:значение засунули в Header , 
 и естественно этотобъект попал в props, и я могу сказать props.name - пропс дай мне имя. Потому, что этот
 объект превращается в props. (И это знание параметра функции. Элементарные знания JS). Мы передаем внутрь 
 функции что-то под одним именем, это объект, и этот же объект преобретает другое имя внутри функции, он 
 становится пропсом. 

 !!!Очень важный момент. Мы вызвали функцию Header , но в реакте мы никогда не вызываем компоненту напрямую.
Мы никогда не ставим скобки вот так Header(obj), это делает реакт сам, самостоятельно. ИМенно он определяет 
какой объект засунутьв Header. Он всегда это делает. 
- Props в компоненту приходит всегда. Реакт всегда засовывает какой-то объект в качестве пропса.
И посути мы просто рисуем тег <Header />
Когда мы рисуем тег <Header /> , ии любой другой, например <Navbar /> , то в него по умолчанию приходит props 
пустой объект. И внутри себя Header не может ничего прочитать потомуЮчто объект пустой. В props пока приходит 
пустой объект, по умолчанию. 
- Компонента Header ждет что ее вызовут и дадут ей пропсы.
const Header = (props) => {
    return (div 
    <a>it-incubator.by</a>
    <a>repetitora.net</a>
    <a>it-kamasutra.net</a>
    div);
}

- И мы можем закинуть в нее вот такой объект: 
{
    name: Dima K,
    age: '30
}
- Как это пироисходит? Мы пишем в разметку тег с данными <Header name='Dima K age='30 /> , формируется объект с этими 
данными и засовывается в props , и наша функция внутри props имеет доступ к этому объекту. 
И она может использовать доступ к данным этого объекта:
{props.name} - и мы увидим Dima K
{props.age} - и мы увидим 30
